## 記録・更新ルール

- 大きな機能追加、広範囲のリファクタリング、ツール更新などは作業単位がまとまった時点で git commit を実施する。
- 更新履歴は、その開発ディレクトリ直下の `<development_dir>/UPDATE.md` で管理する。過去の更新記録に追記する形でアップデート情報を追加すること。バージョン番号は `ver x.x.x` 形式。2 桁目は機能追加、3 桁目は軽微な変更で更新すること。1 桁目はユーザー指示時だけ更新する。
- 依存ライブラリ・外部ツールを追加／削除／置換した際は、ルート `README.md` の `## 使用ライブラリ` の表も更新すること。
